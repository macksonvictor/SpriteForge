# SpriteForge Brand Assets

## Purpose

This package contains the SpriteForge visual identity assets ready to place inside the project.

## Files

### Full logo
Use when there is enough horizontal space, such as README, splash screen, website hero, or documentation.

- `logo_spriteforge_full.png`
- `logo_spriteforge_full_1200w.png`
- `logo_spriteforge_full_800w.png`
- `logo_spriteforge_full_400w.png`

### Wordmark only
Use inside the app header. The user preference is: inside the app, show only the SpriteForge name, not the anvil icon.

- `logo_spriteforge_wordmark.png`
- `logo_spriteforge_wordmark_1200w.png`
- `logo_spriteforge_wordmark_800w.png`
- `logo_spriteforge_wordmark_400w.png`

### Preview
The preview has a dark checker background only to inspect transparency.

- `spriteforge_brand_preview_dark.png`

## Transparency

The PNG files were processed to remove light checkerboard preview backgrounds.
They should have real transparency and should not appear with a white/gray border inside the app.

## Recommended placement

- README top: `assets/brand/logo_spriteforge_full_1200w.png`
- Studio header: `assets/brand/logo_spriteforge_wordmark_400w.png` or text-only title if preferred
- App/window icon: `assets/brand/icon_spriteforge.ico`
- Small UI icon/favicon: `assets/brand/icon_spriteforge_32.png` or `icon_spriteforge_64.png`

## Visual direction

SpriteForge should feel like a retro desktop utility and pixel-art workshop.
Avoid modern AI-glossy branding, glassmorphism, childish mascot styling, and excessive gradients beyond the existing pixel logo colors.
